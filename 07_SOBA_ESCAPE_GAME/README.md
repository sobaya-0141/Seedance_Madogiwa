# そば屋の定時ダッシュ 〜バレずに脱出〜

窓際族の**そば屋**を操作し、定時になったオフィスから**誰にも見つからずに脱出**する
トップダウン型ステルスゲームです。福ちゃん・よーたん・とーくん・やめたろうが巡回し、
おかやまんの監視スクリーンが出口を見張っています。右上のレーダーで各キャラの
**視界（緑のコーン）**を読み、死角を縫って右上の脱出口を目指しましょう。

## 遊び方

- **PC**: `WASD` または矢印キーで移動、`Shift` で「しのび足」（低速だが動きが小さく安全）
- **スマホ**: 左下のバーチャルジョイスティックで移動、右下の「しのび足」ボタン
- 敵の**視界コーン**に入ると画面左上の**発見ゲージ**が上昇。視界から外れると徐々に低下します。
- ゲージが**100%**になる前に隠れること。満タンになると見つかって失敗（リトライ）。
- 壁・机・パーテーションは移動と**視線の両方**を遮ります。物陰を使って進みましょう。
- **右上の脱出口**（緑に光るゲート）に到達すれば定時ダッシュ成功！

## 監視レーダー（右上）

添付イメージのように、フロアの俯瞰図に以下を表示します。

- **白い点**: そば屋（自分）の現在地
- **色付きの点**: 各キャラの現在地（凡例と対応）
- **緑のコーン**: 各キャラの視界範囲。見つかっている間は**赤く点灯**します。
- 壁・什器のアウトライン、脱出口

## 開発

```bash
npm install
npm run dev     # http://localhost:5173 （Vite）
npm run build   # 型チェック + 本番ビルド
```

Vite + TypeScript + Three.js 構成のクライアントのみのアプリで、バックエンドはありません。

## キャラクター3Dモデル（共用ボクセルアセット）

登場キャラのモデルは全ゲーム共用の `../04_GAME_ASSETS/voxel/models/*.glb` を、
`public/models/` からの**相対symlink**で参照しています（モデルはコピーしません）。

- そば屋（プレイヤー） / 福ちゃん / よーたん / とーくん / やめたろう / おかやまん
- 歩行アニメーションは共通の `src/voxel-character-kit.ts`（`05_OFFICE_CRASH_GAME` から流用）で駆動。
  全モデルが標準リグノード（`VoxelRig_*`）を共有しているため、1つのリグ定義で全キャラを動かせます。
- モデルの再生成手順・リグ仕様は `../04_GAME_ASSETS/voxel/README.md` と
  `../04_GAME_ASSETS/voxel/VOXEL_CHARACTER_KIT.md` を参照してください。

## 主なソース構成

- `src/main.ts` — UI・入力（キーボード／タッチ）・オーバーレイのブートストラップ
- `src/game.ts` — シーン構築、モデル読込、更新ループ、視界判定・発見ゲージ、カメラ追従
- `src/level.ts` — オフィスのフロア形状、什器、脱出口、敵の巡回ルートと視界設定（調整用データ）
- `src/radar.ts` — 右上の監視レーダー描画
- `src/geom.ts` — 円と箱の衝突解決、線分と箱の視線判定
- `src/characters.ts` — 各キャラのモデル定義・レーダー色・表示名
- `src/voxel-character-kit.ts` — 共通ボクセルリグ／歩行モーション（共用キット）

## 世界観・禁止事項

`../CLAUDE.md` および `../01_WORLD/WORLD_BIBLE.md` の禁止事項（ブラック企業描写・いじめ・
パワハラ・鬱展開・グロ描写）を厳守しています。本作は「定時で気持ちよく帰る」ための
軽快なかくれんぼで、捕まっても失敗＝リトライのみ（罰・タコ部屋送りの描写はしません）。
各キャラのNG変更（仮面・ギター・ウクレレ・触手など）はモデル側で保持されています。
